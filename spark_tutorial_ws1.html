### Key-value pairs
The `wc` RDD is an RDD of key-value pairs. There are special RDDs in Spark. Normally we would not be able to use `reduceByKey` function on 
just any RDD - only on those consisting of key-value pairs. Some functions again are the same as in Scala (`keys`, `values`, `mapValues`), 
but most are not. They were added in Spark because of the kind of data that Spark deals with, but were not deemed worthy enough of Scala 
Collections. 
Other functions on key-value pairs are 
- **ByKey() functions**: `aggregateByKey`, `combineByKey`, `countByKey`, `foldByKey`, `groupByKey`, `reduceByKey`, `sampleByKey`, 
`subtractByKey`
- **K-V utils**:`keys`, `values`, `lookup`, `mapValues`, `flatMapValues`, `collectAsMap`
- **joins**: `cogroup`, `join`, `leftOuterJoin`, `rightOuterJoin`, `fullOuterJoin`
XXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX

Key-value pairs

The wc RDD is an RDD of key-value pairs. There are special RDDs in Spark. Normally we would not be able to use reduceByKey function on just any RDD - only on those consisting of key-value pairs. Some functions again are the same as in Scala (keys, values, mapValues), but most are not. They were added in Spark because of the kind of data that Spark deals with, but were not deemed worthy enough of Scala Collections.

Other functions on key-value pairs are

  • ByKey() functions: aggregateByKey, combineByKey, countByKey, foldByKey, groupByKey, reduceByKey, sampleByKey, subtractByKey
  • K-V utils:keys, values, lookup, mapValues, flatMapValues, collectAsMap
  • joins: cogroup, join, leftOuterJoin, rightOuterJoin, fullOuterJoin

###Caching
When you want to persist the RDD in memory, you will `cache` it. Any computation will used a memory version. This is where Spark shines. 
If you want to do any analysis on some dataset, you will persist it to memory. 
Remember to free the memory with `unpersist()` when the RDD is not needed any more. Otherwise you might eventually have a "memory leak" 
i.e. find yourself with no memory left. 
XXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX

Caching

When you want to persist the RDD in memory, you will cache it. Any computation will used a memory version. This is where Spark shines. If you want to do any analysis on some dataset, you will persist it to memory.

Remember to free the memory with unpersist() when the RDD is not needed any more. Otherwise you might eventually have a memory leak i.e. find yourself with no memory left.

Tip: You will want to cache an already cleaned RDD - one you will reuse a few times.

### Sorting
- function `sortByKey(Boolean)` - the argument tells us whether to order ascending or descending
- to sort on *values*, swap each pair, sort, then swap each pair again
- if we are interested only in a limited number of ordered values, more efficient option is `takeOrdered(n)(order)` (`order` is optional; 
ascending by default)
- `top(n)` is descending by default
XXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX

Sorting

  • function sortByKey(Boolean) - the argument tells us whether to order ascending or descending
  • to sort on values, swap each pair, sort, then swap each pair again
  • if we are interested only in a limited number of ordered values, more efficient option is takeOrdered(n)(order) (order is optional; ascending by default)
  • top(n) is descending by default
