#### Objectives: 
 - Illustrate **collections** in Scala
 - **Syntax**: especially **defining functions**
 - **transformations**: `map`, `flatMap`, `filter`, `foreach`
 - **aggregations**: `reduce`,  (`fold`, `aggregate`)

### Some points to remember about Scala syntax
- No need for semicolon except to separate statements
- Accessing elements of a collection is 0 based, except for pairs
- **round** brackets is translated to "apply" method of that object; that's why you access elements of an Array with round brackets
- **assignment** operator "=" is translated to "update" method of that object
- Keywords **val, var and def**: 
    - "val" declares immutable values (as in functional programming)  
    - "var" declares mutable variables (as in imperative programming)
    - "def" defines functions
XXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX

Some points to remember about Scala syntax

  • No need for semicolon except to separate statements
  • Accessing elements of a collection is 0 based, except for pairs
  • round brackets is translated to “apply” method of that object; that's why you access elements of an Array with round brackets
  • assignment operator “=” is translated to “update” method of that object
  • Keywords val, var and def:
    • “val” declares immutable values (as in functional programming)
    • “var” declares mutable variables (as in imperative programming)
    • “def” defines functions

#### Applying transformations to collections in Scala
- `map`, `flatMap`, `filter`, `foreach`
- each takes one parameter, and that parameter is a _function_
  - `mylist.map(f)` = apply f to each element of mylist
  - `mylist.flatMap(f)` = apply f to each element of mylist **and flatten** one level
  - `mylist.filter(f)` = **drop** elements x for which f(x) is false
  - `mylist.foreach(f)` = apply f to each element of mylist, **return nothing**
- remember that Scala syntax says we can omit dot and parenthesis
XXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX

Applying transformations to collections in Scala

  • map, flatMap, filter, foreach
  • each takes one parameter, and that parameter is a function
  • mylist.map(f) = apply f to each element of mylist
  • mylist.flatMap(f) = apply f to each element of mylist and flatten one level
  • mylist.filter(f) = drop elements x for which f(x) is false
  • mylist.foreach(f) = apply f to each element of mylist, return nothing
  • remember that Scala syntax says we can omit dot and parenthesis

- if anonymous function has only one parameter, you can use underscore `_`
- you can declare partially applied anonymous functions using `case`
